Angular Rails API
  • 📤PDF uploaded via Admin → Form Builder UI

🛡️ PHI Risk: Uploaded forms may contain real patient names, DOBs, phone numbers. Our pipeline never retains these values — they are stripped in the next stage.

🧹 PHI Removed
Aspose.PDF flattens the document — all field values and interactive elements are erased
The Aspose JAR clears all filled values, flattens every AcroForm field into the static page content, and outputs a clean "Blank Form Template." No PHI survives this step.
Aspose.PDF JAR Rails Controller
  • 🔧Aspose clears every field value, then calls field.flatten()
  • 🛡️PHI wiped: John Smith → (empty), 03/15/1985 → (empty)
  • 📄Result: static PDF — looks like the original but with no interactive fields
  • 💾Flattened PDF stored in form_templates.flattened_pdf
  • 📊Page dimensions captured (width, height in PDF points)
🛡️ 7-Layer PHI Defense: Field values cleared → Fields flattened → Textract values discarded → No PHI in field mappings → No PHI in generated template → Admin UI never shows patient data → Database stores only blank templates.

PDF.js Fabric.js Angular
  • 📐PDF.js renders the flattened PDF pixel-perfect in a <canvas>
  • 🎨Fabric.js canvas sits on top — supports drag, drop, resize
  • 🔍HiDPI scaling (Device Pixel Ratio) for crisp text on retina displays
  • 📏Canvas coordinates ↔ PDF points conversion for accurate placement
  • 📄Multi-page navigation with page selector

🤖 AI + OCR
Dual-path detection with OCR — fields auto-mapped to Surgimate VARs
PATH A uses Aspose for structured AcroForm fields. PATH B uses Textract OCR to scan labels and layout — even on scanned or image-based PDFs. Detected fields are auto-mapped to Surgimate VARs.
Aspose — PATH A Textract OCR — PATH B Fabric.js Canvas VarMatchingService
  • 🅰️PATH A (Aspose): Detects AcroForm fields — instant, zero cost
  • 🅱️PATH B (Textract OCR): Fallback — OCR reads labels, detects KEY-VALUE pairs & checkboxes
  • 🔗Auto-Mapping: "First Name" → PatientFirstName (96% confidence)
  • 🟢Green ≥ 80% Orange ≥ 50% Red < 50% Blue = Manual
  • ✏️Users can accept, reject, re-map, resize, or add fields manually
  • 💾Saved as JSON in form_templates.field_mappings

✅ Filled & Ready
Aspose merges event data into the template — print, download, or save as attachment
When a user prints or downloads from the surgical event, the system pulls patient/event VARs as XML, matches them to field mappings, and Aspose writes values into the flattened PDF.
Aspose.PDF JAR Rails Controller Angular Print Tab
  • 📋VARs XML: event.to_document_xml generates patient & event data
  • 🔧Aspose reads field_mappings JSON, looks up each VAR in XML, writes values at x/y coordinates
  • 🧮Calculated fields: Formulas like {PatientLastName} - {PatientFirstName} resolved at fill time
  • 📎Save as Attachment: Optionally stores filled PDF in the event's attachments tab
  • 🖨️Output: inline preview, download, or print — all from the same endpoint
